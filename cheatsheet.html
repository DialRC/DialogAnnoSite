Dialog Performance Evaluation Manual

Recommendation Rewards
  • Label every system utterance with the recommendation intent with binary reward {-1, 1}.
  • A system utterance is a recommendation if it matches the following templates.
    • (yn)request-food
    • (yn)request-cu
    • (yn)request-mr_clue
  • A recommendation gets 1 point if the users agree with the recommendation, otherwise -1
Entrance Rewards
  • Every system utterance that switches to a new agent should be labelled for entrance reward
  • If the transition is correct +1 point otherwise -1 point
Exit Rewards
  • when an session with remote agent finishes, the whole sub-dialog should be labelled for exit reward
  • For Cambridge/Yelp/Weather reward = success - number turns, where success = +/- 5
  • For Mr Clue, the reward = number of turns
  • For chatbot, the reward is +/- 1 for good response and bad response
